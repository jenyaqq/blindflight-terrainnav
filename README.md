# BlindFlight TerrainNav PRO

Гражданская система резервной навигации лёгких воздушных судов по цифровой модели рельефа и данным радиовысотомера.

Система работает по принципу terrain-referenced navigation:

```text
H_terrain = H_abs - H_radio
```

По барометрической высоте и радиовысоте восстанавливается профиль рельефа под воздушным судном. Затем программа перебирает возможные азимуты, скорости и смещения на DEM-карте, строит эталонные профили и выбирает траекторию с максимальной нормированной корреляцией.

## Возможности

- Главный экран Navigation Solution: текущая позиция, координаты карты, скорость, азимут, вектор скорости и доверие.
- 3D Terrain Matching: DEM-сцена с веером проверяемых направлений, альтернативными кандидатами, лучшим совпадением, текущей позицией и зоной ошибки.
- Отображение геометрии корреляционного поиска во время расчёта.
- Terrain fingerprint: сравнение профиля из радиовысотомера с профилем DEM вдоль найденного решения.
- Оценочный радиус ошибки `R≈... м` с разложением на вклад сетки, азимута, скорости, корреляции, неоднозначности и информативности рельефа.
- Проверка качества NMEA: количество строк, GGA, checksum OK/BAD, частота измерений.
- Экспорт CSV, JSON и GeoJSON для интеграции с GIS-системами.
- Пресеты поиска: **Проверка** для третьего чекпоинта, быстрый режим, режим азимута 1°, точный режим.
- Скользящее окно обработки профиля для поэтапного обновления навигационного решения.

## Реализованный навигационный цикл

- загрузка реальной DEM-карты GeoTIFF через Rasterio;
- генерация синтетической DEM без внешних данных;
- формирование контрольного профиля для проверки точности;
- генерация NMEA-0183 GGA-подобного потока радиовысотомера с checksum;
- загрузка готового NMEA-файла;
- восстановление профиля рельефа из радиовысотомера;
- корреляционный поиск траектории;
- автоматический поиск скорости;
- поиск азимута и смещения;
- опциональный перебор стартовой точки вокруг центра карты;
- опциональный фильтр Калмана для шумного профиля;
- оценка достоверности результата;
- расчёт ошибки в метрах в контрольном режиме;
- визуализация DEM, маршрута, профилей, heatmap и 3D Terrain Matching;
- выгрузка найденной траектории CSV;
- выгрузка маршрута GeoJSON;
- выгрузка отчёта JSON.

## Установка и запуск

### Windows PowerShell

```powershell
cd terrain_nav_streamlit
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

### Linux / macOS

```bash
cd terrain_nav_streamlit
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

После запуска Streamlit откроет веб-интерфейс в браузере.

## Быстрый тест без интерфейса

```bash
python smoke_test.py
```

Ожидаемый результат: найденный азимут около `38°`, скорость около `40 м/с`, корреляция около `0.99`, средняя ошибка меньше `100 м`.

## Проверка через UI

Оставьте параметры по умолчанию:

```text
Режим: Контрольный профиль
Пресет поиска: Быстрый режим
DEM: Синтетическая DEM
Размер пикселя: 30 м/пиксель
Контрольная скорость: 40 м/с
Контрольный азимут: 38°
Shift: 40 px
Шум: 2 м
Искать скорость автоматически: да
Скорости: 20...80 м/с, шаг 5 м/с
```

Нажмите **«Сформировать профиль и найти траекторию»**. Во время расчёта появится 3D Terrain Matching с активной областью поиска, после завершения — найденная текущая позиция, скорость, азимут, корреляция, оценка ошибки, NMEA-статистика и вкладка с 3D-сценой terrain matching.

## Режим азимута 1°

Для полного перебора направлений выберите пресет:

```text
Азимут 1°
```

Этот режим использует шаг перебора азимута `1°`. Если расчёт на ноутбуке идёт тяжело, можно использовать быстрый режим для первичной оценки, а затем переключиться на режим азимута `1°` для уточнения.

## Работа с реальной DEM

Рекомендуемый пайплайн через QGIS:

1. Скачать DEM: SRTM / Copernicus GLO-30 / ALOS AW3D30.
2. Открыть DEM в QGIS.
3. Выбрать участок с выраженным рельефом: горы, холмы, пересечённая местность.
4. Обрезать участок примерно 20×20 км или 30×30 км.
5. Перепроецировать слой в UTM, чтобы координаты и пиксели были в метрах.
6. Экспортировать GeoTIFF.
7. Загрузить GeoTIFF в приложение.
8. В поле **«Размер пикселя для расчёта»** поставить реальное разрешение DEM, например `30 м/пиксель`.

Если GeoTIFF остаётся в широте/долготе, приложение всё равно попробует работать, но скорость и ошибки в метрах будут менее надёжными.

## Сценарии применения

- доставка медикаментов, продуктов и оборудования в труднодоступные районы;
- снабжение арктических метеостанций и геологоразведочных лагерей;
- резервная навигация гражданских воздушных судов при деградации GNSS;
- восстановление фактического маршрута после миссии по записи радиовысотомера.

## Ограничения

- на плоском рельефе профиль менее информативен;
- качество DEM влияет на точность;
- для потокового режима алгоритм запускается на скользящем окне последних измерений;
- система является резервным навигационным каналом и должна использоваться вместе с другими бортовыми источниками навигационной информации.

### Вариант 2. Через CLI без интерфейса

```bash
python checkpoint_runner.py \
  --dem data/map.tif \
  --heights data/heights.txt \
  --start-x 250 \
  --start-y 250 \
  --coord-system px \
  --azimuth 38 \
  --dt 0.5 \
  --height-kind radio \
  --max-shift 0 \
  --speed-min 20 --speed-max 80 --speed-step 1 \
  --out checkpoint_output
```

Если `(x, y)` даны в координатах карты GeoTIFF, используйте:

```bash
--coord-system map
```

Результаты CLI:

```text
checkpoint_output/
├── estimated_route.csv
├── estimated_route.geojson
├── navigation_report.json
├── trajectory_on_dem.png
└── correlation_heatmap.png
```

В отчёте сохраняются локальные координаты `x_px/y_px`, координаты карты `map_x/map_y`, lon/lat при наличии CRS, скорость, азимут, компоненты скорости east/north, корреляция и оценка ошибки.
